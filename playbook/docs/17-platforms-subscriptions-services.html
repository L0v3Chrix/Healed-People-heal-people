Organization-owned, never personal

No campaign, finance, or program-critical account lives on a personal login. Every platform has an organization owner and a backup — because the community's work should never depend on one person's inbox.

Open every account the same careful way

Account setup standard

One checklist, applied to every account, so nothing critical is discoverable only in somebody's memory.

For every account, record all sixteen of these:

👤Ownership

  • Legal account name
  • Organization owner
  • Billing owner
  • Backup admin

🔑Access & security

  • MFA
  • Recovery method
  • Roles
  • Incident contact

📦Data & terms

  • Data processed
  • Processor/subprocessor terms

💳Money

  • Production tier
  • Spending cap
  • Renewal date

🚪The way out

  • Export format
  • Offboarding procedure
  • Deletion confirmation
Credential rule — no exceptions

No shared credential may be stored in source, chat, or a personal notes file.

What we're deliberately not signing up for yet

Services intentionally deferred

Saying "not yet" to these keeps the beta focused, the budget honest, and the data footprint small.

  • POS integrations
  • CRM/ad-network profile export
  • Automated payout
  • SMS
  • AI personalization
  • Data warehouse
  • Native mobile app distribution
  • International shipping
  • Multi-city operations
How a deferred service earns its way in

Add only after a documented workflow, data review, budget, owner, test plan, and vendor-exit path are approved.

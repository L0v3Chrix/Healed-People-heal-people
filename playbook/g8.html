The decision

Continue, revise, pause, or stop — the evidence decides, and everyone hears it.

20 tasks Gated

Executive / product 14 tasks

  • Publish the impact report with only approved aggregate evidenceGated

    Produce the beta impact report containing only claims-approved aggregate evidence — no individual stories, generated people, or illustrative numbers.

    ♥ The first public proof of impact must be as honest as the program claims to be.

    Done when: The impact report is published with every figure traced to approved aggregate evidence in the claims register. · source

  • Synthesize the beta evidence reportGated

    Synthesize the beta research and operational data into the evidence report that grounds the exit decision.

    ♥ A decision this consequential deserves evidence, not anecdotes.

    Done when: The evidence report is complete, covering the beta-success criteria with supporting data. · source

  • Make the continue/revise/pause/stop exit decisionGated

    Hold the exit review — desired by December 21, 2026, fallback January 2027 — and decide continue, revise, pause, or stop based on the evidence report and remediation backlog.

    ♥ Participants, partners, and investors all deserve a decision made on evidence rather than momentum.

    Done when: A documented continue/revise/pause/stop decision exists, referencing the evidence report and remediation backlog. · source

  • Decide whether to ever add safe impersonationGated

    Beta ships with no impersonation; after the pilot, decide whether a safe impersonation capability is justified and, if so, under what controls.

    ♥ Protects participants from anyone acting as them until the need and safeguards are proven.

    Done when: A documented decision (approve with controls, or keep prohibition) is recorded with reasoning. · source

  • Revisit deferred AI, POS, and payout-provider integrationsGated

    Decision checkpoint: production AI, POS integrations, and Stripe Connect/payout providers are deferred until participant-value classification, identity, and custody are approved — decide after the pilot whether to proceed.

    ♥ Complexity and custody risk are only taken on once the pilot proves they're worth it.

    Done when: A documented post-pilot decision exists for each deferred integration with reasoning. · source

  • Revisit deferred native mobile apps and custom Express APIGated

    Decision checkpoint: native mobile apps and a custom Express API are deferred because responsive web with partner-assisted access covers the approved beta and Next.js server routes are sufficient at pilot scale — decide after the pilot whether either is now justified.

    ♥ Participants get the simplest platform that serves them until real evidence says more is needed.

    Done when: A documented post-pilot decision exists for native mobile apps and for a custom API layer, each with reasoning. · source

  • Prepare the pilot close report and continue/revise/pause/stop memoGated

    Write the pilot close report and the decision memo recommending continue, revise, pause, or stop.

    ♥ The team owes itself and its investors an honest verdict on what the pilot proved.

    Done when: Both documents are complete and the decision is recorded. · source

  • Create the public impact report template using aggregate evidence onlyGated

    Design a public impact report format that uses only aggregate, privacy-safe evidence.

    ♥ Impact should be shown without ever putting an individual recipient on display.

    Done when: The template is approved through the claims register workflow. · source

  • Run the decision checkpoint for intentionally deferred servicesGated

    Reconsider POS, CRM/ad-network export, automated payout, SMS, AI personalization, data warehouse, native mobile distribution, international shipping, and multi-city operations — adding any only after a documented workflow, data review, budget, owner, test plan, and vendor-exit path are approved.

    ♥ Deliberate restraint keeps the pilot focused; this checkpoint keeps the restraint deliberate rather than forgotten.

    Done when: A written decision exists for each deferred service, and any adopted service has all six approval artifacts. · source

  • Reconsider Patreon membership and Indiegogo InDemand after the betaGated

    ♥ Recurring supporters deserve a considered offer built on a proven program, not a membership sold before there's anything ongoing to belong to.

    Done when: A dated decision record exists for both options, made after the beta against actual drop and campaign outcomes. · source

  • Produce beta evidence reportGated
